SEEMA FRONTEND API CALLS - EXECUTIVE SUMMARY
=============================================

ANALYSIS SCOPE:
- 35 page.tsx files analyzed
- 4 custom hooks (useApi, useApiMutation, useApiList)
- 1 auth store (Zustand) with API calls
- 122+ unique endpoint paths
- Organized by business domain

KEY FINDINGS:

1. ENDPOINTS BY CATEGORY:
   - Authentication (6 endpoints): login, register, refresh, change-password, sessions, logout
   - Admin/Firm (21 endpoints): user management, firm settings, email, preferences, imports
   - Compliance (52+ endpoints): alerts, checks, remediation, evidence, audit trails
   - AML/Sanctions (6 endpoints): CDD, SAR, PEP screening, sanctions checks
   - Accounts & Reconciliation (8 endpoints): accounts, transactions, reconciliations
   - Conflicts (5 endpoints): conflict checks, party management, resolution
   - Breaches/Incidents (2 endpoints): breach reports
   - Complaints (5 endpoints): complaints, acknowledgment, resolution
   - Staff & Training (3 endpoints): staff, training completion, action logging
   - Supervision (4 endpoints): supervision scheduling and completion
   - Matters & Checklists (3 endpoints): matter checklists, item completion
   - Policies (3 endpoints): policy management, generation
   - Audit Reports (2 endpoints): report generation and listing
   - SRA Returns (3 endpoints): SRA return data, export
   - Key Dates (5 endpoints): limitation periods, CPR, pre-action protocols
   - Undertakings (3 endpoints): undertakings, fulfill, breach
   - Billing (6 endpoints): subscription, history, payment methods
   - Onboarding (2 endpoints): SRA lookup, completion
   - Integrations (5 endpoints): Clio OAuth, sync, status

2. REQUEST METHODS BREAKDOWN:
   - GET:    ~60 calls (data retrieval, filtering)
   - POST:   ~45 calls (creation, actions, state changes)
   - PUT:    ~8 calls (updates, edits)
   - DELETE: ~4 calls (user deletion, payment method removal)
   - PATCH:  0 calls (not used)

3. COMMON PATTERNS:
   - Query Parameters: page, per_page, status, sort
   - Path Parameters: ${id}, ${userId}, ${recordId}
   - Request Bodies: JSON with typed fields
   - Responses: Envelope pattern with { data, message?, success? }
   - Error Responses: { message, code?, details?, status? }

4. AUTHENTICATION MODEL:
   - Bearer token in Authorization header
   - Tokens stored in localStorage (accessToken, refreshToken, user)
   - Token refresh auto-handled by axios interceptor
   - Demo mode fallback via demo-token

5. DATA SHAPES:
   - User: id, email, name, role, firm_id, firm_name, avatar?, last_login?
   - StaffMember: id, firm_id, name, email, role, job_title?, department?, status, training_progress?
   - ComplianceAlert: id, title, alert_type, severity, status, description, created_at
   - ClientAccount: id, client_name, matter_ref, balance, status, residual, last_reconciled, next_recon_due
   - CDD Record: id, client_name, client_type, cdd_level, risk_level, status, verifications
   - SAR Record: id, client_name, matter_ref, suspicion_type, amount, status, mlro_decision

6. KEY WORKFLOWS:
   - Login → Authenticate → Get Tokens → Store → Redirect
   - Compliance Check → View Results → Create Remediation → Track Actions
   - AML Onboarding → CDD Creation → PEP/Sanctions Screening → Verify
   - Account Management → Open Account → Record Transactions → Reconcile → COFA Signoff
   - Conflict Check → Run Check → Resolve (with waiver if needed)

FILES CONTAINING API CALLS:
- src/lib/api.ts (core axios client with auth interceptor)
- src/lib/api-client.ts (functional wrapper for stores)
- src/lib/hooks/use-api.ts (useApi, useApiMutation, useApiList hooks)
- src/lib/stores/auth-store.ts (login/refresh in Zustand)
- 35 page modules in src/app/*/page.tsx

CRITICAL IMPLEMENTATION DETAILS:
1. All API requests go through axios with retry logic
2. 401 errors trigger token refresh with request queuing
3. Response envelope always has { data, message?, success? }
4. FormData used for file uploads (no special headers needed)
5. Query params for pagination: page, per_page
6. Path params use template literals: `/path/${variable}`
7. Demo mode check: localStorage.getItem('accessToken') === 'demo-token'
